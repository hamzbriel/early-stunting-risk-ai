# Dataset

## Overview

The dataset used in this project is **fully synthetic**, generated by the Synthetic Data Platform located in `synthetic_data/`. It contains **10,000 records** representing children under 5 years old, with 20 features spanning demographic, socioeconomic, nutritional, and healthcare dimensions.

> **Important**: This is not real patient data. The dataset is built from probability distributions and dependency rules grounded in public health literature to produce realistic patterns for demonstration and development purposes.

## Feature Groups

### 1. Child Information

| Feature | Type | Values / Range | Description |
|---------|------|----------------|-------------|
| `age_month` | Integer | 0 - 59 | Child age in months (under 5 years) |
| `gender` | Categorical | `M`, `F` | Child biological sex |
| `birth_weight` | Float | 1.5 - 5.0 kg | Birth weight (normally distributed, μ=3.1, σ=0.45) |
| `birth_length` | Float | 40.0 - 60.0 cm | Birth length (normally distributed, μ=49.5, σ=2.5) |

### 2. Mother Information

| Feature | Type | Values / Range | Description |
|---------|------|----------------|-------------|
| `mother_age` | Integer | 15 - 50 | Mother's age in years |
| `mother_education` | Categorical | `none`, `primary`, `secondary`, `higher` | Highest education level |
| `mother_working` | Binary | 0, 1 | Employment status (0=no, 1=yes) |

### 3. Father Information

| Feature | Type | Values / Range | Description |
|---------|------|----------------|-------------|
| `father_education` | Categorical | `none`, `primary`, `secondary`, `higher` | Highest education level |
| `father_working` | Binary | 0, 1 | Employment status (0=no, 1=yes) |

### 4. Household Information

| Feature | Type | Values / Range | Description |
|---------|------|----------------|-------------|
| `family_income` | Categorical | `very_low`, `low`, `medium`, `high` | Monthly income tier |
| `sanitation` | Categorical | `poor`, `fair`, `good` | Sanitation facility quality |
| `clean_water` | Binary | 0, 1 | Access to clean drinking water |
| `electricity` | Binary | 0, 1 | Access to electricity |
| `house_density` | Categorical | `low`, `medium`, `high` | Household occupancy density |

### 5. Nutrition Information

| Feature | Type | Values / Range | Description |
|---------|------|----------------|-------------|
| `exclusive_breastfeeding` | Binary | 0, 1 | Exclusive breastfeeding for first 6 months |
| `protein_intake` | Categorical | `low`, `medium`, `high` | Daily protein intake adequacy |
| `vitamin_intake` | Categorical | `low`, `medium`, `high` | Micronutrient intake adequacy |

### 6. Healthcare Information

| Feature | Type | Values / Range | Description |
|---------|------|----------------|-------------|
| `immunization` | Categorical | `none`, `partial`, `complete` | Immunization completeness |
| `diarrhea_history` | Binary | 0, 1 | History of diarrhea in past 3 months |
| `healthcare_access` | Categorical | `poor`, `fair`, `good` | Access to healthcare facilities |

## Target Variable

| Column | Type | Values | Description |
|--------|------|--------|-------------|
| `risk_score` | Float | 0 - 100 | Continuous stunting risk score |
| `risk_level` | Categorical | `Low`, `Medium`, `High` | Discretized risk classification |

### Expected Distribution

| Risk Level | Proportion |
|------------|-----------|
| Low | ~35% |
| Medium | ~40% |
| High | ~25% |

## Feature Dependencies

The dataset models realistic inter-feature correlations through a Directed Acyclic Graph (DAG). Key relationships include:

### Positive Correlations (risk factors cluster together)

- Low income → poor sanitation → no clean water → diarrhea
- Low education → low income → poor nutrition → high stunting risk
- Low birth weight → low birth length (correlation = 0.75)

### Protective Clustering

- Higher education → higher income → better nutrition → complete immunization → lower risk

### No Direct Causal Relationships

- Gender has minimal direct effect on stunting risk
- Father's working status has indirect effects mainly through income

## Data Split

The dataset is split into three subsets, stratified by `risk_level`:

| Split | Records | Purpose |
|-------|---------|---------|
| Training | 7,000 (70%) | Model training |
| Validation | 1,500 (15%) | Hyperparameter tuning |
| Test | 1,500 (15%) | Final evaluation |

## Feature Engineering (Preprocessing)

Before model training, the raw features undergo the following transformations:

- **Numerical features**: Standard scaling (z-score normalization)
- **Categorical features**: One-hot encoding
- **Target encoding**: `risk_level` is label-encoded (Low→0, Medium→1, High→2)

The processed feature space expands from 20 original features to **42 engineering features** after one-hot encoding and preprocessing.
